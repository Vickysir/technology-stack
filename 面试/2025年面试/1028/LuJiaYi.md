- 请解释 JavaScript 的事件循环机制

  - JavaScript 是单线程的，事件循环是用来处理异步操作的一种执行模型
  - 事件循环组成部分
    - 执行栈（后进先出），代码运行时，函数被压入栈中，执行完毕后移除
    - 任务队列，储存异步函数和回调函数
      - 微任务（Promise.catch、MutationObserver）
      - 宏任务（setTimeout、DOM 事件）
      - 微任务 > 宏任务
    - 事件循环
      - 一个持续的循环过程，负责协调执行栈和任务队列。它会不断检查执行栈，并在执行栈为空时，将任务队列中的任务推入执行栈执行。
  - 执行流程
    - 执行同步代码
    - 处理异步任务
    - 异步回调入队
    - 事件循环开始（同步任务处理完成）
    - 优先处理微任务（清空微任务队列）
    - 从宏任务队列中取出一个任务，执行宏任务（先进先出）
    - 循环往复
      - 在宏任务执行过程中，如果产生了新的微任务或宏任务，它们会继续被添加到各自的队列中
      - 事件循环会再次执行微任务，然后再执行宏任务，如此循环往复，直到所有任务都被处理完毕。
