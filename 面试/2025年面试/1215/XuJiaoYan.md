## 四、算法题

### 题目：LeetCode 394. 字符串解码（Decode String）

**题目链接：** https://leetcode.cn/problems/decode-string/

**题目背景：**
基于候选人在前端开发中处理模板解析、配置处理的经验（如Figma D2C插件中的代码转换、组件库的配置解析等），选择一道与实际工作场景相关的算法题。

**题目描述：**
给定一个经过编码的字符串，返回它解码后的字符串。

编码规则为：`k[encoded_string]`，表示其中方括号内部的 `encoded_string` 正好重复 `k` 次。注意 `k` 保证为正整数。

你可以认为输入字符串总是有效的；输入字符串中没有额外的空格，且输入的方括号总是符合格式要求的。

此外，你可以认为原始数据不包含数字，所有的数字只表示重复的次数 `k` ，例如不会出现像 `3a` 或 `2[4]` 的输入。

**示例 1：**
```
输入：s = "3[a]2[bc]"
输出："aaabcbc"
```

**示例 2：**
```
输入：s = "3[a2[c]]"
输出："accaccacc"
```

**示例 3：**
```
输入：s = "2[abc]3[cd]ef"
输出："abcabccdcdcdef"
```

**示例 4：**
```
输入：s = "abc3[cd]xyz"
输出："abccdcdcdxyz"
```

**提示：**
- `1 <= s.length <= 30`
- `s` 由小写英文字母、数字和方括号 `'[]'` 组成
- `s` 保证是一个有效的输入
- 所有整数都在 `[1, 300]` 的范围内

**评分标准：**
- **基础实现（60分）**：能正确解析一层嵌套的字符串（如 `3[a]2[bc]`）
- **嵌套处理（30分）**：能正确处理多层嵌套（如 `3[a2[c]]`）
- **代码质量（10分）**：代码可读性、边界情况处理、时间复杂度分析

**考察点：**
1. **算法能力**：
   - 栈（Stack）的使用：处理嵌套结构
   - 递归思想：递归解析嵌套字符串
   - 字符串处理：数字解析、字符串拼接

2. **前端相关场景**：
   - 模板解析（类似Vue/React模板编译）
   - 配置解析（组件库配置、设计规范解析）
   - 代码转换（Figma D2C插件中的代码生成）

3. **代码能力**：
   - 边界情况处理（空字符串、单个字符等）
   - 代码可读性和注释
   - 时间复杂度分析（O(n)）

**参考实现思路：**

**方法一：栈（推荐）**
```typescript
function decodeString(s: string): string {
  const stack: Array<string | number> = [];
  let num = 0;
  let res = '';
  
  for (let char of s) {
    if (char >= '0' && char <= '9') {
      // 处理多位数
      num = num * 10 + parseInt(char);
    } else if (char === '[') {
      // 将当前结果和数字入栈
      stack.push(res);
      stack.push(num);
      res = '';
      num = 0;
    } else if (char === ']') {
      // 出栈，重复字符串
      const repeatCount = stack.pop() as number;
      const prevRes = stack.pop() as string;
      res = prevRes + res.repeat(repeatCount);
    } else {
      // 普通字符，直接拼接
      res += char;
    }
  }
  
  return res;
}
```

**方法二：递归**
```typescript
function decodeString(s: string): string {
  let index = 0;
  
  function dfs(): string {
    let res = '';
    let num = 0;
    
    while (index < s.length) {
      const char = s[index];
      
      if (char >= '0' && char <= '9') {
        num = num * 10 + parseInt(char);
        index++;
      } else if (char === '[') {
        index++; // 跳过 '['
        const innerStr = dfs();
        res += innerStr.repeat(num);
        num = 0;
      } else if (char === ']') {
        index++; // 跳过 ']'
        return res;
      } else {
        res += char;
        index++;
      }
    }
    
    return res;
  }
  
  return dfs();
}
```

**时间复杂度：** O(n)，其中 n 是解码后字符串的长度  
**空间复杂度：** O(n)，栈或递归调用栈的空间

**预期讨论：**
1. **算法选择**：为什么选择栈而不是递归？各自的优缺点？
2. **边界情况**：如何处理空字符串、单个字符、纯数字等情况？
3. **性能优化**：如果字符串很长，如何优化？
4. **实际应用**：

   - 与配置解析的结合（组件库配置、设计规范）
   - 与代码生成的结合（Figma D2C插件中的代码转换）
